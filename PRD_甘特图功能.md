## 1. 项目背景

现有的项目管理/任务排期系统已支持任务的录入、编辑和导出功能。但在排期管理与跟踪方面，当前仅提供列表/表格视图，缺乏直观的时间线可视化能力。

项目管理人员和一线执行人员需要通过甘特图快速掌握：
- 所有有排期但尚未上线的任务的整体进度
- 任务之间的时间重叠与依赖关系
- 受周末/节假日影响的实际工作日排布

## 2. 产品目标

在现有页面的右上角操作区，"导出"按钮之后新增一个「甘特图」切换按钮，点击后页面从列表视图切换到甘特图视图。甘特图仅展示「有排期计划但尚未实际上线」的任务，横轴为日期时间线，周末与节假日用特殊颜色标注。

## 3. 功能范围

### 3.1 本期包含

| 功能模块 | 描述 |
|----------|------|
| 甘特图视图切换 | 右上角新增「甘特图」按钮，点击切换视图 |
| 甘特图渲染 | 基于任务排期数据渲染甘特条 |
| 日期横轴 | 支持按日/周维度的日期时间线展示 |
| 周末/节假日标注 | 周末与节假日用特殊背景色高亮标记 |
| 排期过滤 | 仅展示「有计划排期但无实际上线时间」的任务 |
| 视图切换回列表 | 再次点击或关闭甘特图视图，返回列表视图 |

### 3.2 本期不包含

| 功能 | 说明 |
|------|------|
| 甘特图内拖拽修改排期 | 本期仅展示，不允许直接在甘特图上拖拽修改 |
| 任务依赖连线 | 任务间的前后依赖关系连线本期暂不实现 |
| 资源负载视图 | 按人员维度的资源负载分析本期暂不实现 |
| 甘特图导出 | 甘特图视图的图片/PDF导出能力本期暂不实现 |

## 4. 用户角色

| 角色 | 描述 | 核心诉求 |
|------|------|----------|
| 项目管理人员 | 负责项目整体排期与进度跟踪 | 快速了解所有未上线任务的时间分布与进度 |
| 任务执行人员 | 负责具体任务的执行 | 查看自己负责任务的时间排布 |
| 管理者/领导 | 宏观了解项目进展 | 一览所有排期任务与时间节点 |

## 5. 功能需求详述

### 5.1 入口：视图切换按钮

**位置：** 页面右上角操作栏，在「录入」和「导出」按钮之后。

**交互：**
- 默认状态：显示「甘特图」文字按钮（或图标+文字）
- 点击后：当前视图切换为甘特图视图，按钮变为「列表视图」
- 再次点击：切换回列表视图
- 状态保持：切换仅在当前页面会话中生效，刷新后默认回到列表视图

**视觉：**
- 按钮风格与「录入」「导出」保持一致
- 可使用甘特图/Gantt 图标的 + 「甘特图」文字标签

```
[录入] [导出] [甘特图]   ← 列表视图时
[录入] [导出] [列表视图]  ← 甘特图视图时
```

### 5.2 甘特图视图

#### 5.2.1 数据范围

- **过滤条件：** 任务具有「计划开始时间」和「计划结束时间」（即有排期），且「实际上线时间」为空（未上线）
- **排序规则：** 按「计划开始时间」升序排列
- **数据加载：** 进入甘特图视图时，前端请求后端甘特图专用接口获取数据

#### 5.2.2 日期横轴（X轴）

- **默认范围：** 从最早任务的计划开始时间往前3天，到最晚任务的计划结束时间往后7天
- **刻度单位：** 按日显示，支持水平滚动
- **日期格式：** `MM/DD`（或 `MM-DD`），鼠标悬停显示完整日期 `YYYY-MM-DD`
- **当日标记：** 在当前日期位置显示一条竖直红色虚线（今日线）

#### 5.2.3 周末与节假日标注

- **周末：** 周六和周日，背景色使用浅灰色（`#F0F0F0`）或浅蓝色（`#EBF5FB`）
- **节假日：** 按照国家法定节假日，背景色使用浅红色（`#FDEDEC`）或浅橙色（`#FEF5E7`）
- **标注范围：** X轴顶部的日期区域背景色变化，同时对应的任务条区域背景同步变化
- **图例：** 甘特图左上角或右上角显示颜色图例：「周末 | 节假日」

#### 5.2.4 任务行（Y轴）

- **行高：** 固定行高（建议 36px ~ 40px），保证可读性
- **任务条：** 每个任务以横向条状展示，从左（开始时间）到右（结束时间）
- **任务条颜色：** 按任务状态使用不同颜色：
  - 进行中（进行中）：蓝色 `#5B9BD5`
  - 未开始：灰色 `#A6A6A6`
  - 已逾期（当前时间 > 计划结束时间）：红色 `#FF6B6B`
- **任务条文字：** 条内显示任务名称（超出用...省略），鼠标悬停显示完整信息（参见 5.2.5）
- **任务分组：** 可按项目/分类进行分组，分组标题行固定且不可展开折叠（本期简单分组）

#### 5.2.5 交互细节

| 交互 | 行为 |
|------|------|
| 鼠标悬停任务条 | 弹出 Tooltip，显示：任务名称、负责人、计划开始时间、计划结束时间、状态、所属项目 |
| 鼠标滚轮 | 水平滚动日期区域（若内容超出可视区） |
| 点击任务条 | 跳转/打开该任务的详情编辑弹窗或详情页 |
| 横向拖拽 | 平移整个甘特图的时间窗口 |

#### 5.2.6 空状态

- 当没有符合条件（有排期未上线）的任务时：
  - 显示空状态插画 + 文字「暂无排期中未上线的任务」
  - 同时显示「录入任务」快捷按钮

#### 5.2.7 加载状态

- 数据加载中显示骨架屏或 Loading 动画
- 加载失败显示错误提示 + 「重新加载」按钮

## 6. 数据模型

### 6.1 后端接口

**接口定义：** `GET /api/gantt/tasks`

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| projectId | Long | 否 | 项目ID，为空则查询所有项目 |
| startDate | String | 否 | 排期开始日期范围过滤（yyyy-MM-dd） |
| endDate | String | 否 | 排期结束日期范围过滤（yyyy-MM-dd） |

**响应格式：**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "projects": [
      {
        "projectId": 1,
        "projectName": "XXX项目",
        "tasks": [
          {
            "taskId": 100,
            "taskName": "需求评审",
            "assignee": "张三",
            "planStartDate": "2026-06-01",
            "planEndDate": "2026-06-05",
            "actualLaunchDate": null,
            "status": "IN_PROGRESS",
            "priority": "HIGH"
          }
        ]
      }
    ],
    "holidays": ["2026-06-08", "2026-06-09"]
  }
}
```

### 6.2 核心字段说明

| 字段 | 说明 | 备注 |
|------|------|------|
| planStartDate | 计划开始时间 | 有值即为"有排期" |
| planEndDate | 计划结束时间 | 有值即为"有排期" |
| actualLaunchDate | 实际上线/完成时间 | 为空表示"未上线" |
| status | 任务状态 | ENUM: NOT_STARTED, IN_PROGRESS, COMPLETED, DELAYED |

### 6.3 节假日数据

- **方案：** 后端维护年度节假日配置表，或调用第三方节假日 API
- **缓存：** 节假日数据可缓存到 Redis，减少重复计算
- **接口：** 同甘特图数据接口一并返回当前展示时间范围内的节假日列表
- **前台判定：** 前端根据接口返回的节假日列表 + 周末（周六/日）自动进行颜色标注

## 7. 非功能需求

### 7.1 性能要求

| 指标 | 要求 |
|------|------|
| 甘特图渲染时间 | 1000条任务内，页面加载到渲染完成 ≤ 2s |
| 接口响应时间 | 95% 的请求响应 ≤ 500ms |
| 水平滚动流畅度 | 60fps，无明显卡顿 |

### 7.2 兼容性

- 支持 Chrome 90+、Edge 90+、Firefox 90+ 最新两个大版本
- 支持 1920x1080 及以上分辨率，适配 1366x768 最低分辨率
- 暂不支持移动端甘特图视图

### 7.3 缓存策略

- 甘特图数据：Redis 缓存 5 分钟，任务状态变更时主动失效
- 节假日数据：Redis 缓存 1 天，因为节假日配置变动极低
- 前端缓存：不缓存甘特图数据，每次切换视图时重新请求

## 8. 技术方案建议

### 8.1 前端技术选型

## 9. 备忘录功能

### 9.1 功能概述

在工具栏「录入」按钮左侧新增两个按钮，提供每日备忘录的录入与查询功能。

### 9.2 入口：工具栏按钮

**位置：** 页面右上角操作栏，在「录入」按钮之前。

**两个按钮：**
- **📝 当日备忘** — 绿色边框按钮，点击打开当日备忘编辑弹窗
- **📋 查看备忘** — 橙色边框按钮，点击打开备忘历史记录弹窗

```
[📝 当日备忘] [📋 查看备忘] [+ 录入] [导出] [甘特图]
```

### 9.3 当日备忘弹窗

**标题：** `当日备忘 - YYYY-MM-DD`（默认当天日期）

**交互逻辑：**
- 顶部输入框 + 「添加」按钮，输入事项后回车或点击按钮添加到列表
- 每个事项左侧有复选框，勾选后该事项文字添加中划线（标记完成）
- 每个事项右侧有 × 删除按钮，鼠标悬停时显示
- 双击事项文字可快速切换完成状态
- 点击弹窗遮罩层或「取消」按钮关闭弹窗

**数据存储：**
- 每天的备忘录独立存储，按日期唯一
- 内容以 JSON 格式存储：`[{"text": "事项内容", "done": true/false}]`
- 点击「保存」按钮将当前内容持久化到数据库
- 保存后自动关闭弹窗

**每日清空逻辑：**
- 进入弹窗时自动加载当天已有数据
- 每天首次打开时内容为空（数据库无当天记录时返回空列表）
- 数据由用户手动清理或覆盖保存

### 9.4 查看备忘弹窗

**标题：** 备忘历史记录

**交互逻辑：**
- 展示所有已保存的备忘记录，按日期倒序排列（最新在前）
- 每条记录显示日期 + 星期，点击展开查看详情
- 展开后显示该日所有事项，已完成事项显示 ✅ 标记和中划线
- 点击遮罩层或「关闭」按钮关闭弹窗

### 9.5 空状态

- 当日备忘无事项时显示"暂无事项，请在上方输入添加"
- 备忘历史无记录时显示"暂无备忘记录"

### 9.6 数据模型

#### 9.6.1 后端接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/memos/today?date=2026-06-17` | 获取指定日期的备忘 |
| GET | `/api/memos/history` | 获取全部备忘历史（按日期倒序） |
| POST | `/api/memos` | 保存备忘（有则更新，无则新增） |

**保存请求体：**
```json
{
  "memoDate": "2026-06-17",
  "content": "[{\"text\": \"完成需求评审\", \"done\": false}, {\"text\": \"写测试用例\", \"done\": true}]"
}
```

#### 9.6.2 数据库表

```sql
CREATE TABLE `memo` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '主键',
  `memo_date` date NOT NULL COMMENT '备忘日期',
  `content` json NOT NULL COMMENT '备忘内容(JSON: [{text, done}])',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_memo_date` (`memo_date`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='每日备忘录表';
```

#### 9.6.3 核心字段说明

| 字段 | 说明 | 备注 |
|------|------|------|
| memo_date | 备忘日期 | UNIQUE 约束，每天仅一条记录 |
| content | 备忘内容 JSON 字符串 | `[{text: string, done: boolean}]` |

## 10. 链接收藏功能

### 10.1 功能概述

在工具栏「当日备忘」按钮左侧新增「链接收藏」按钮，提供常用链接的收藏与管理功能。

### 10.2 入口：工具栏按钮

**位置：** 页面右上角操作栏，在「当日备忘」按钮之前。

**按钮样式：**
- **🔗 链接收藏** — 粉色边框按钮，点击打开链接收藏弹窗

```
[🔗 链接收藏] [📝 当日备忘] [📋 查看备忘] [+ 录入] [导出] [甘特图]
```

### 10.3 链接收藏弹窗

**标题：** 🔗 链接收藏

**交互逻辑：**
- 弹窗顶部提供表单输入区：名称、链接地址、优先级
- 点击「添加」按钮保存链接，保存后自动刷新列表
- 列表中每个链接显示优先级数字、名称、链接地址
- 每个链接右侧有三个操作按钮：打开链接（新窗口）、编辑、删除
- 点击「编辑」后表单切换为编辑模式，按钮变为「更新」
- 点击遮罩层或「关闭」按钮关闭弹窗

**排序逻辑：**
- 按优先级数字升序排列，数字越小越靠前
- 同优先级按创建时间倒序

### 10.4 空状态

- 无链接时显示"暂无收藏链接，在上方添加吧"

### 10.5 数据模型

#### 10.5.1 后端接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/links` | 获取所有链接（按优先级排序） |
| POST | `/api/links` | 保存链接（新增/更新，有ID则更新） |
| DELETE | `/api/links/{id}` | 删除链接 |

**保存请求体：**
```json
{
  "id": null,
  "name": "Jira 看板",
  "url": "https://jira.example.com",
  "priority": 1
}
```

#### 10.5.2 数据库表

```sql
CREATE TABLE `link_item` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '主键',
  `name` varchar(200) NOT NULL COMMENT '链接名称',
  `url` varchar(1000) NOT NULL COMMENT '链接地址',
  `priority` int NOT NULL DEFAULT 0 COMMENT '优先级排序(数字越小越靠前)',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  PRIMARY KEY (`id`),
  KEY `idx_priority` (`priority`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='链接收藏表';
```

### 10.6 核心字段说明

| 字段 | 说明 | 备注 |
|------|------|------|
| name | 链接名称 | 必填，用于显示 |
| url | 链接地址 | 必填，点击打开的新窗口地址 |
| priority | 优先级 | 数字越小排序越靠前 |

## 11. 部署方案

### 11.1 前后端合并部署

Spring Boot 后端可直接打包前端构建产物，生成一个独立可运行的 JAR 文件，无需单独启动前端开发服务器。

**构建方式：**
1. 前端构建：`cd frontend && npm run build`，产物输出到 `backend/src/main/resources/static/`
2. 后端打包：`cd backend && mvn package`，生成 `gantt-chart-1.0.0.jar`
3. 启动：`java -jar backend/target/gantt-chart-1.0.0.jar`

**关键配置：**
`WebConfig.java` 实现了 SPA 路由 fallback，确保所有前端路由在直接刷新时返回 `index.html`。

### 11.2 Windows 服务部署

利用 `ganttchart-service.xml` 可将后端注册为 Windows 服务，实现开机自启、崩溃自动重启、日志轮转。

**注册服务：**
```bash
ganttchart-service.exe install
```

**服务属性：**
- 名称：`GanttChart`
- 启动模式：自动（延迟启动）
- 失败恢复：自动重启（10秒/30秒延迟）
- 日志路径：`backend/logs/`
- 超时停止：15秒
